# SSH Config Sync from ECS Instances

## Overview

The SSH sync system automatically discovers EC2 instances behind ECS services and writes their IPs into your SSH config. Instead of manually looking up IPs with `get_all_ips.py` and typing `ssh ec2-user@10.0.0.5`, you can simply run:

```bash
ssh wordpress-staging-nginx
```

A systemd user timer keeps the config current by re-syncing every 15 minutes. No root access is required.

## How It Works

```
┌──────────────┐     ┌──────────────┐     ┌────────────────────┐
│  systemd     │────>│  sync_ssh.py │────>│  AWS ECS/EC2 APIs  │
│  user timer  │     │              │     │  (via boto3)       │
│  (15 min)    │     │  reads:      │     └────────────────────┘
└──────────────┘     │  ~/.aws/     │              │
                     │  credentials │              │
                     │  [mesh]      │     IPs returned per
                     │              │     cluster/service
                     │  writes:     │              │
                     │  ~/.ssh/     │<─────────────┘
                     │  ecs_hosts   │
                     └──────────────┘
```

1. **Discovery:** `sync_ssh.py` calls the AWS ECS API to list all clusters and services, then resolves the underlying EC2 instance IPs via the EC2 API.

2. **Config generation:** For each cluster/service pair, it creates an SSH host alias named `{cluster}-{service}` with the first IP address. The full output is written to `~/.ssh/ecs_hosts`.

3. **SSH integration:** An `Include ~/.ssh/ecs_hosts` directive at the top of `~/.ssh/config` makes the generated hosts available to SSH, SCP, rsync, and any tool that reads SSH config.

4. **Freshness:** The file is fully regenerated on each run. Stale entries (e.g., terminated instances) are automatically removed.

### Example generated config

After a sync, `~/.ssh/ecs_hosts` might contain:

```
# Auto-generated by sync_ssh.py -- do not edit manually

Host kcworks-dev-nginx
  HostName 10.0.0.5
  User ec2-user

Host kcworks-production-nginx
  HostName 10.0.1.12
  User ec2-user

Host kcworks-staging-wordpress
  HostName 10.0.2.8
  User ec2-user
```

## Prerequisites

- **Python 3.12+**
- **[uv](https://docs.astral.sh/uv/)** - Python package manager (handles dependencies automatically)
- **AWS credentials** with permission to call:
  - `ecs:ListClusters`, `ecs:ListServices`, `ecs:ListTasks`, `ecs:DescribeTasks`, `ecs:DescribeContainerInstances`
  - `ec2:DescribeInstances`
- **[1Password CLI](https://developer.1password.com/docs/cli/)** (`op`) - only needed once, during initial setup, to pull AWS credentials from your vault
- **systemd** (user mode) - present on all modern Linux distributions
- **Network access** to the ECS instances (e.g., via VPN) if using private IPs

## File Layout

| File | Purpose |
|------|---------|
| `ecs_utils.py` | Shared module: queries ECS/EC2 and returns `{(cluster, service): [ip, ...]}` |
| `sync_ssh.py` | CLI script that generates SSH config from ECS instance data |
| `setup_aws_profile.sh` | Pulls AWS credentials from 1Password into `~/.aws/credentials` |
| `install.sh` | Full setup: credentials, systemd units, initial sync |
| `uninstall.sh` | Tears down timer, symlinks, generated SSH config |
| `sync_now.sh` | Triggers an immediate re-sync outside the timer schedule |
| `.env.tpl` | 1Password secret reference URIs (used by `setup_aws_profile.sh`) |
| `systemd/sync-ssh-ecs.service` | systemd oneshot unit for the sync |
| `systemd/sync-ssh-ecs.timer` | systemd timer (fires every 15 minutes) |
| `get_all_ips.py` | Standalone CLI to print all ECS instance IPs (uses `ecs_utils`) |
| `get_ip.py` | Standalone CLI to print IPs for a specific cluster (uses `ecs_utils`) |

## Setup

### 1. Install

From the project directory, run:

```bash
./install.sh
```

This does everything in one step:

1. Prompts 1Password to read your AWS credentials (one-time interactive approval)
2. Writes them to `~/.aws/credentials` under the `[mesh]` profile
3. Generates systemd unit files with absolute paths matching your repo location
4. Symlinks the units into `~/.config/systemd/user/`
5. Enables and starts the 15-minute timer
6. Adds `Include ~/.ssh/ecs_hosts` to the top of `~/.ssh/config` (idempotent)
7. Runs an initial sync so hosts are available immediately

After install, verify with:

```bash
# Check the timer is active
systemctl --user status sync-ssh-ecs.timer

# Check the generated hosts
cat ~/.ssh/ecs_hosts

# Test SSH (replace with an actual host from the output above)
ssh kcworks-dev-nginx
```

### 2. Verify

```bash
# Dry run: see what would be generated without writing anything
uv run sync_ssh.py --profile mesh --dry-run

# Check SSH config integration
head -1 ~/.ssh/config
# Should show: Include /home/<you>/.ssh/ecs_hosts
```

## Day-to-Day Usage

### SSH to an ECS instance

```bash
ssh kcworks-staging-nginx
```

The host alias follows the pattern `{cluster}-{service}`. Tab completion works if your shell supports it.

### Force an immediate re-sync

```bash
./sync_now.sh
```

This accepts the same options as `sync_ssh.py`:

```bash
./sync_now.sh --dry-run          # preview without writing
./sync_now.sh --public           # use public IPs instead of private
./sync_now.sh --user ubuntu      # override the SSH user
./sync_now.sh --identity-file ~/.ssh/mykey.pem  # specify an SSH key
```

### Check timer status

```bash
systemctl --user status sync-ssh-ecs.timer
systemctl --user list-timers      # shows next scheduled run
```

### View sync logs

```bash
journalctl --user -u sync-ssh-ecs.service --since today
```

## Credential Rotation

When AWS credentials are rotated, re-run:

```bash
./setup_aws_profile.sh
```

This reads the new values from 1Password and overwrites the `[mesh]` profile in `~/.aws/credentials`. The timer picks up the new credentials on its next run. No restart needed.

To use a different profile name:

```bash
./setup_aws_profile.sh myprofile
```

## Customisation

### Change the sync interval

Edit `systemd/sync-ssh-ecs.timer` and change `OnUnitActiveSec`:

```ini
[Timer]
OnBootSec=1min
OnUnitActiveSec=30min   # change from 15min to 30min
```

Then re-run `./install.sh` to regenerate and reload the units.

### Use public IPs

Edit `systemd/sync-ssh-ecs.service` and add `--public` to the `ExecStart` line, or re-run `install.sh` after modifying it to pass `--public`.

### Specify an SSH key

Add `--identity-file ~/.ssh/mykey.pem` to the `ExecStart` line in the service unit. Each generated host entry will then include an `IdentityFile` directive.

### Change the SSH user

Add `--user ubuntu` to the `ExecStart` line.

## Uninstall

```bash
./uninstall.sh
```

This will:

1. Stop and disable the systemd timer
2. Remove the unit symlinks from `~/.config/systemd/user/`
3. Remove the `Include` directive from `~/.ssh/config`
4. Delete `~/.ssh/ecs_hosts`

The `[mesh]` profile in `~/.aws/credentials` is left in place. Remove it manually if no longer needed:

```bash
# Edit ~/.aws/credentials and delete the [mesh] section
```

## Troubleshooting

### Timer isn't running

```bash
# Check if the timer is enabled
systemctl --user is-enabled sync-ssh-ecs.timer

# If not, re-run install
./install.sh

# Check for errors in the service
journalctl --user -u sync-ssh-ecs.service -n 20
```

### No hosts generated

```bash
# Verify AWS credentials work
aws sts get-caller-identity --profile mesh

# Run a dry run to see what's discovered
uv run sync_ssh.py --profile mesh --dry-run

# Check if there are any ECS clusters
aws ecs list-clusters --profile mesh
```

### SSH still asks for IP / host not found

- Check that `~/.ssh/config` starts with `Include ~/.ssh/ecs_hosts`
- Check that `~/.ssh/ecs_hosts` contains the expected host
- Ensure you're using the exact alias: `{cluster}-{service}` (e.g., `kcworks-dev-nginx`)

### Moved the repo to a different directory

Re-run `./install.sh`. It regenerates the systemd units with the correct absolute paths and is safe to run multiple times.

## Architecture

The shared module `ecs_utils.py` contains the core logic for querying ECS and EC2. Both the standalone IP scripts (`get_ip.py`, `get_all_ips.py`) and the SSH sync script (`sync_ssh.py`) import from it:

```
ecs_utils.py                 # shared: get_cluster_service_ips()
├── get_ip.py                # CLI: single cluster
├── get_all_ips.py           # CLI: all clusters
└── sync_ssh.py              # CLI: write SSH config
```

The function signature:

```python
def get_cluster_service_ips(
    region: str,
    profile: str | None,
    cluster_name: str | None,
    service: str | None,
    private: bool,
) -> dict[tuple[str, str], list[str]]
```

Returns a dict mapping `(cluster_name, service_name)` tuples to lists of IP addresses.
